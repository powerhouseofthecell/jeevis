BFS - Breadth First Search

Traversal Algorithm

Given a starting node, find its children, then explore each of them before exploring their children. Below is a randomly generated graph. If you click one of the nodes, you will see BFS performed in slow motion.

How it works

Starting at an arbitrary node, BFS goes by "breadth," meaning that the algorithm adds each of the node's children to its list of nodes to search (often a queue). It then pulls the next node from this queue or list and does the same. It runs until this list of nodes to check is empty.

What is it used for?

One of the major uses is for answering the question of "what nodes are visitable from this node?" The list of all visited nodes at the completion of the algorithm answers this question. Additionally, with a small modification to the original algorithm, it is easy to determine a path between any 2 nodes and further the length of this path.

Path

So BFS can tell you if there exists a path between nodes, but is this useful if you don't know the path itself? While yes, it is useful to know merely the existence of a path, if we imagine a scenario where if there exists a path, then we also want to know the path itself, how would you modify the algorithm outlined above? How could you keep track of the path as the algorithm runs?
